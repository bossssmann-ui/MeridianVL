# SEO — задачи по видимости сайта в поиске и для ИИ

Цель: сайт активно индексируется в рунете (Яндекс — приоритет, Google — вторым), выдаётся по коммерческим и информационным запросам, и при этом полностью «читаем» для ИИ-систем (YandexGPT/Алиса, ChatGPT, Perplexity, Claude), которые всё чаще отвечают на вопросы «где арендовать авто во Владивостоке» вместо классической выдачи.

## 0. Архитектурное требование (критично, решает всё остальное)

⚠️ В HANDOFF заложен рендер каталога и маршрутов из JSON на клиенте. Для SEO и ИИ это слабое место: Яндекс и Google исполняют JS с задержками, а большинство ИИ-краулеров (GPTBot, ClaudeBot, PerplexityBot) **не исполняют JavaScript вообще** — они увидят пустые страницы.

- [ ] **Весь смысловой контент должен присутствовать в HTML сразу**, без ожидания JS. Решение для нашей статики: простой build-скрипт (Node.js, ~50 строк) генерирует готовые HTML-страницы из JSON-данных перед публикацией. JS остаётся только для фильтров, калькулятора и интерактива.
- [ ] Каждый маршрут — **отдельный физический HTML-файл** с ЧПУ: `/marshruty/ostrov-russkiy/`, а не `route.html?id=russky`. То же для карточек авто: `/avto/toyota-rav4/`.
- [ ] Проверка: открыть страницу с отключённым JS — весь текст, цены и описания должны быть видны.

## 1. Техническая база индексации

- [ ] `robots.txt`: открыть сайт всем, **явно разрешить ИИ-ботов** — YandexBot, Googlebot, GPTBot, ClaudeBot, Claude-Web, PerplexityBot, Amazonbot, CCBot (Common Crawl — из него учатся многие модели).
- [ ] `sitemap.xml` с автогенерацией при сборке (все страницы: маршруты, авто, FAQ), указать в robots.txt и в панелях вебмастеров.
- [ ] `llms.txt` в корне сайта — краткое машиночитаемое описание: кто мы, город, услуги, парк, условия, контакты, ссылки на ключевые страницы. Новый стандарт для ИИ-краулеров, у конкурентов его нет.
- [ ] Canonical на всех страницах, корректные 404, редирект www→без www, единый протокол HTTPS.
- [ ] Кириллические ЧПУ не использовать — только латиница-транслит.
- [ ] Favicon + разметка для сниппетов (Яндекс показывает фавикон в выдаче).

## 2. Регистрация и панели (первая неделя после публикации)

- [ ] **Яндекс.Вебмастер**: добавить сайт, подтвердить, назначить регион «Владивосток» (критично для локальной выдачи), отправить sitemap, включить обход по счётчику Метрики.
- [ ] **Яндекс Бизнес** — карточка организации: адрес, часы, фото, категория «Прокат автомобилей». Карточка попадает в Карты и колдунщик локальной выдачи — для запроса «аренда авто владивосток» это половина трафика.
- [ ] **2ГИС** — обязательная регистрация, во Владивостоке им пользуются активнее Карт.
- [ ] Google Search Console + Google Business Profile.
- [ ] Яндекс.Метрика + цели (заявка, клик по WhatsApp/телефону); вебвизор для анализа поведения.

## 3. Структурированные данные (Schema.org, JSON-LD)

Разметка — главный канал, через который и поисковики, и ИИ понимают сайт «без чтения».

- [ ] Главная: `LocalBusiness` (подтип `AutoRental`) — название, адрес, гео-координаты, телефон, часы, район обслуживания, рейтинг.
- [ ] Карточки авто: `Product` + `Offer` (цена от, валюта, наличие) — даёт цену прямо в сниппете выдачи.
- [ ] FAQ: `FAQPage` — вопросы-ответы попадают в расширенный сниппет и почти дословно цитируются ИИ-ассистентами.
- [ ] Маршруты: `Article` + `TouristTrip` (точки, длительность) + `BreadcrumbList`.
- [ ] События из афиши: `Event` (даты, место) — шанс попасть в событийные колдунщики Яндекса.
- [ ] Отзывы: `AggregateRating` (подтягивать рейтинг с Яндекс Карт вручную, честно).
- [ ] Валидация: Яндекс Вебмастер → «Проверка микроразметки» + validator.schema.org.

## 4. Семантика и структура контента

- [ ] Собрать семантическое ядро (Яндекс.Вордстат по региону Владивосток): коммерческие («аренда авто владивосток», «прокат авто владивосток недорого», «аренда авто владивосток без водителя», «прокат внедорожника владивосток», «аренда авто аэропорт кневичи»), информационные («куда съездить из владивостока», «что посмотреть в приморье на машине», «остров русский на машине»).
- [ ] Каждая страница = один запросный кластер: главная — общий коммерческий; страницы классов авто («аренда кроссовера во Владивостоке») — под средне­частотники; маршруты — под информационные длинные хвосты, которые конкурент не собирает.
- [ ] Заголовки h1–h3 строго по иерархии, один h1 на страницу, ключ в h1/title/description без переспама (Яндекс наказывает фильтром «Баден-Баден» за переоптимизацию).
- [ ] Тексты писать для людей, вопросно-ответным форматом где уместно — такие блоки ИИ цитируют чаще всего.
- [ ] Alt-тексты всех фото («Toyota RAV4 2021 в аренду во Владивостоке», «маяк Токаревского — маршрут на арендованном авто»).
- [ ] Перелинковка: маршрут ↔ рекомендованные авто ↔ класс ↔ FAQ; хлебные крошки на всех страницах.

## 5. Скорость и мобильность (фактор ранжирования)

- [ ] Core Web Vitals: LCP < 2,5 c, CLS < 0,1. Наша статика без фреймворков даёт это почти бесплатно — не испортить тяжёлыми фото.
- [ ] Фото: WebP/AVIF, ленивая загрузка ниже первого экрана, `width/height` у всех `<img>`, размеры под ретину без избыточности.
- [ ] Шрифты: `font-display: swap`, предзагрузка, подмножество кириллицы.
- [ ] Хостинг с точкой присутствия в РФ (для Владивостока — важно; проверить TTFB из ДВ-региона).
- [ ] Lighthouse mobile ≥ 90 по всем метрикам — контроль на каждом этапе, не в конце.

## 6. E-E-A-T и доверие (важно и для Яндекса, и для ИИ)

- [ ] Полные реквизиты (ИП/ООО, ИНН, ОГРН) в подвале — Яндекс ранжирует коммерческие сайты с реквизитами выше.
- [ ] Страница «О компании»: реальные фото офиса и парка, история, команда.
- [ ] Реальные отзывы: виджет Яндекс Карт + стимулировать гостей оставлять отзывы (QR-код при возврате авто).
- [ ] Единый NAP (название-адрес-телефон) везде: сайт, Яндекс Бизнес, 2ГИС, соцсети. Разнобой ломает локальное ранжирование и путает ИИ.
- [ ] Страницы политики ПДн и согласия (уже в плане) — тоже сигнал доверия.

## 7. Внешние сигналы и упоминания

- [ ] Профили: VK, Telegram-канал, Дзен (статьи-маршруты дублировать в Дзен со ссылкой — бесплатный трафик рунета).
- [ ] Регистрация в тематических каталогах: visit-primorye.ru (туристический портал края), vl.ru, справочники проката.
- [ ] Партнёрские ссылки (см. PLACES.md, раздел 6): базы отдыха и активности ссылаются на нас — естественная региональная ссылочная масса.
- [ ] Пресс-повод для местных СМИ (vl.ru, primpress): «сервис публикует бесплатные автомаршруты по Приморью с картой запретных зон» — инфоповод, которого нет у конкурентов.

## 8. Видимость для ИИ (GEO — generative engine optimization)

Сводка того, что из списков выше работает на ИИ, + специфика:

- [ ] Контент в чистом HTML без JS-зависимости (раздел 0) — базовое условие.
- [ ] llms.txt + разрешённые ИИ-боты в robots.txt (раздел 1).
- [ ] FAQ и маршруты в формате «вопрос → прямой ответ в первом абзаце» — ИИ берут готовые формулировки.
- [ ] Факты о компании одинаковы во всех источниках (сайт, Яндекс Бизнес, 2ГИС, соцсети) — ИИ сверяют источники, противоречия снижают шанс попасть в ответ.
- [ ] На ключевых страницах — короткий факт-блок: «MeridianVL — прокат автомобилей во Владивостоке. Парк N авто от X ₽/сутки. Подача в аэропорт Кневичи. Работаем с ГГГГ года» — идеальная цитата для ИИ-ответа.
- [ ] Раз в квартал проверять руками: спросить у Алисы, ChatGPT и Perplexity «где арендовать авто во Владивостоке» и смотреть, попадаем ли в ответ; фиксировать динамику.
